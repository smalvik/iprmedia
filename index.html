Предисловие: анализ экосистемы IPR Media

Перед выполнением задания я изучил оба ключевых продукта компании — IPR SMART (для ВУЗов) и PROFобразование (для СПО) — как два продукта одной экосистемы. Ряд решений из IPR SMART можно адаптировать для PROF, не изобретая с нуля:

Возможность IPR SMART (ВУЗы) IPR PROF (СПО) Потенциал
SMART-курсы Есть (интерактивные курсы с тестами) Нет — только тесты Высокий
Образовательные модули Есть (структурированные блоки) Нет Высокий

Вывод: IPR SMART ушёл дальше в интерактивности (курсы, модули, лекторий, интеграции). PROF сильнее в специализации под СПО (лаборатории, структура по профессиям). Стратегия: перенести проверенные механики из SMART, сохранив уникальную специализацию PROF.

Сценарий 1

«Рабочий стол преподавателя» — ежедневный дашборд

Проблема: преподаватель заходит в ЭБС только чтобы «найти книгу». Нет рабочего пространства.

Решение: персональный дашборд, который агрегирует:

  • Мои дисциплины — список предметов с привязанной литературой (автоподбор по ФГОС + ручная корректировка)
  • Задания студентам — назначает главы к прочтению → студент получает задание в ЛК → преподаватель видит статистику
  • Мои группы — статистика: книговыдачи, глубина чтения, активность
  • Новинки по дисциплинам — персонализированная лента обновлений

Книговыдачи: каждое назначение = 20-30 книговыдач (число студентов). Один преподаватель ведёт 3-5 дисциплин × 2-4 группы = 200-600 книговыдач/семестр.

Сценарий 2

«Учебный трек студента» — персонализированный путь обучения

Проблема: студент не знает, что искать. Каталог из 17 500+ изданий перегружает.

Решение: при первом входе студент указывает специальность и курс. Платформа формирует:

  • Мой учебный план — литература по текущим дисциплинам
  • Задания от преподавателей — назначенные материалы с дедлайнами
  • Рекомендации — «Студенты твоей специальности также читают...»
  • Подготовка к экзаменам — автоматические подборки перед сессией
  • Прогресс — визуальный трекер прочитанного

Книговыдачи: персонализация повышает конверсию. Вместо «ищи в 17 500 изданий» → «вот 15 книг по 5 предметам». Переход Dormant → Occasional = +50% книговыдач.

Сценарий 3

«Методический конструктор» — сборка РПД и списков литературы

Проблема: преподаватели тратят часы на формирование списков литературы для РПД — вручную проверяя актуальность изданий.

Решение: конструктор, который:

  • Выбор специальности → дисциплины → автоматический подбор изданий из каталога
  • Проверка актуальности: год издания, гриф, соответствие ФГОС
  • Готовый блок «Список литературы» в формате РПД
  • Уведомления о новых изданиях по дисциплинам

Книговыдачи: каждый список = регулярные обращения к изданиям. Это системная привязка контента к учебному процессу.

Логика сборки контента вокруг УМК

УМК в СПО — пакет документов по дисциплине: РПД, ФОС, методические указания, список литературы. Сегодня преподаватель собирает это вручную. Идея: сделать УМК «живой оболочкой» вокруг контента на платформе.

Состав УМК на платформе:

  • Основная литература (автоподбор по ФГОС)
  • Дополнительная литература (ручной выбор)
  • Практические задания (банк кейсов, тесты)
  • Методические указания (загрузка преподавателем)
  • ФОС — фонд оценочных средств
  • Индустриальные кейсы от партнёров

Принцип: «Начни с малого — получи большое»

Преподаватель не должен «создавать УМК» — это пугает. Вместо этого:

  1. 1 минута: Выбрать специальность и дисциплину → платформа автоматически формирует подборку
  2. 5 минут: Скорректировать — убрать лишнее, добавить своё
  3. Опционально: Добавить тесты, прикрепить методические указания, назначить студентам
  4. Результат: «список литературы» естественно вырастает в полноценный УМК

Решение 1

Банк практических кейсов от индустриальных партнёров

IPR Media уже наращивает сеть партнёров и аккумулирует практикоориентированные кейсы. Нужно встроить их в учебный процесс.

Формат кейса на платформе:

  • Отрасль + партнёр (бренд компании)
  • Описание ситуации (реальная задача из практики)
  • Задание студенту
  • Материалы для подготовки — ссылки на издания из каталога (генерирует книговыдачи!)
  • Критерии оценки + экспертный разбор

Связь с Профессионалитетом: проект требует интеграции работодателей — банк кейсов = готовый инструмент для колледжей-участников.

Интеграция с экосистемой: кейсы из PROF → входная точка в проектную работу на DataLIB.Проекты.

Решение 2

Практические задания с проверкой

Расширение существующего раздела тестов (369 тестов):

  • Тесты по главам — привязаны к разделам изданий → стимул прочитать
  • Ситуационные задания — мини-кейсы с вариантами решений
  • Задания с развёрнутым ответом — студент привязывает цитату из издания

Ключевое: каждое задание требует обращения к контенту платформы → органический рост книговыдач.

Решение 3

Цифровые стажировки и профессиональные симуляции

Модульные мини-курсы, моделирующие рабочий день специалиста:

  • «День бухгалтера» — типовые задачи с профильной литературой
  • «Приёмка на строительном объекте» — ситуационные задания с нормативной документацией
  • Привязка к 200+ специальностям каталога PROF

Аналог из IPR SMART: SMART-курсы — интерактивный формат уже разработан для ВУЗов, адаптировать под практику СПО.

Текущая модель и её ограничение

Сегодня: централизованная подписка от министерства → колледж получает доступ → использование минимальное → продление «по инерции».

Проблема: продление не привязано к ценности. Решение о покупке принимает не пользователь. При сокращении бюджетов первыми отлетают «формальные» подписки.

Сценарий 2 — Целевой

«Пакет Профессионалитет»

Проект «Профессионалитет» = кластерная модель, колледж + работодатель. Требуется практикоориентированный контент.

  • Базовая подписка: контент + подборки
  • + Пакет «Профессионалитет»: банк кейсов + цифровые стажировки + аналитика для работодателя
  • + Пакет «Методическая поддержка»: конструктор УМК + отчёты для аккредитации + IPR Education

Ценообразование: модульная модель → начинают с базы, наращивают по мере использования.

Гипотеза 1

«Нет проводника»

Продукт продаётся руководству (B2G), но никто не обучает конечных пользователей.

Источник: институциональные отзывы PROFобразование — обратная связь только от руководства, не от конечных пользователей.

Проверка: спросить у неактивных — знают ли про ЭБС, проходили ли обучение.

Гипотеза 2

«Нет интеграции в рабочий процесс»

ЭБС существует отдельно. Преподаватель использует Word для РПД, Moodle для заданий.

Источник: отсутствие отзывов о рабочих сценариях — пользователи воспринимают ЭБС как ридер, а не как рабочий инструмент.

Проверка: карта инструментов преподавателя — где ЭБС, а где могла бы быть.

Гипотеза 3

«Мобильный ридер — главный барьер для студентов»

Неудобная навигация, выделение текста вместо скролла, проблемы с PDF — студенты бросают и уходят в Google.

Источник: реальные отзывы Google Play IPR SMART (3.9★).

Проверка: аналитика мобильных сессий, bounce rate, A/B тест нового ридера. Потенциал: +10-15 NPS pts.

Гипотеза 4

«Аудио и текст живут отдельно»

PROF имеет 1 100+ аудиоматериалов, но нет синхронизации с текстом. Студент не может переключиться с чтения на аудио.

Источник: B2C стандарт — Audible «Read & Listen», LitRes, Tribooks.

Проверка: пилот на 10 изданиях с аудио+текст sync. Потенциал: рост времени на платформе ×2-3.

Гипотеза 5

«Конкурентное преимущество стабильности не используется»

IPR SMART стабильнее Лани в 2 раза по рейтингу (3.9★ vs 1.9★), но это не транслируется в маркетинг.

Источник: Google Play, RuStore — публичные рейтинги.

Проверка: включить сравнительные рейтинги в sales-материалы, отследить конверсию продления.

Подход к оценке и росту NPS

Текущий NPS неизвестен — публичных данных нет. Предложение:

  • Измерение: in-app NPS-опрос после 5+ книговыдач
  • Сегментация: преподаватели / студенты / библиотекари — у каждого свой NPS
  • Proxy: Google Play 3.9★ ≈ NPS +10..+20
  • Рычаги роста: мобильный ридер (H3) → +10-15 pts, синхронизация (H4) → +5-8 pts, AI-саммаризация → рост activation

Подготовка

  4. Параллельно: инфраструктура нового стека + backend

Параллельная разработка + миграция

Итоговый вопрос
Через 3 года IPR ПРОФ — это...

Через 3 года IPR ПРОФ — это цифровая методическая среда, в которой преподаватель колледжа проводит значительную часть рабочего дня: формирует учебные программы, назначает материалы студентам, создаёт практические задания на основе реальных кейсов от работодателей, отслеживает прогресс групп. Для студента это персональное учебное пространство от первого курса до выпуска — с учебниками, практикой, AI-помощником и портфолио выполненных проектов.

Продукт перестал быть «электронной библиотекой для аккредитации» и стал инструментом, без которого колледж не представляет организацию учебного процесса — как не представляет работу без электронного журнала.

Ключевой сдвиг: решение о продлении подписки больше не принимается «по инерции» или «для галочки». Оно принимается потому, что 40%+ преподавателей ведут в PROF свои дисциплины, студенты ежедневно открывают платформу, а руководство получает прозрачную аналитику качества образования. IPR ПРОФ — стандартная инфраструктура СПО, такая же обязательная и привычная, как расписание занятий.
